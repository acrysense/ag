# Подключение данных в таблицу — пошагово

Один способ, без режимов: **бэк отдаёт один JSON со всеми строками, фронт сам строит
таблицу, сортировку, фильтрацию и пагинацию в браузере.** Подходит для наших таблиц
(сотрудники, аптеки, менеджеры — до нескольких тысяч строк).

> Проверено: 2000 строк × 13 колонок грузятся за ~40 мс, сортировка ~20 мс,
> переход по страницам ~10 мс. В DOM всегда только текущая страница — не виснет.

---

## Шаг 1. Поставить секцию таблицы на страницу

В шаблоне страницы вставить секцию с атрибутом `data-table-src` — ссылкой на ваш PHP,
который отдаёт JSON:

```html
<section class="data-panel employees-table"
         data-data-table
         data-module="DataTable"
         data-path="components"
         data-table-src="/local/ajax/employees.php"></section>
```

Всё. Заголовки, тело таблицы, контролы (сортировка/вид), пагинацию фронт построит сам
из JSON. **Своими руками `<thead>`/`<tbody>` писать не нужно.**

> Заголовок панели (`<h1>`) и кнопку «Экспорт» при желании можно оставить внутри секции
> руками — фронт дописывает таблицу после них.

---

## Шаг 2. PHP отдаёт JSON

Готовый прокомментированный пример: **[`datatable-config.php`](datatable-config.php)**.
Скопируйте его, замените заглушку на реальную выборку — и всё.

Файл должен:
- отдавать `Content-Type: application/json; charset=utf-8`;
- быть доступен по URL из `data-table-src` (200, **без HTTP-авторизации** на этом пути);
- возвращать **все строки** (без LIMIT).

---

## Формат JSON (контракт)

```jsonc
{
  "columns": [ /* описание колонок, порядок = порядок колонок */ ],
  "rows":    [ /* все строки; ключи строки = column.key */ ],
  "pageSize":  20,            // размер страницы по умолчанию
  "pageSizes": [20, 50, 100], // варианты в селекте "показывать по"
  "total":     2000           // = count(rows)
}
```

### Колонка (`columns[]`)

| поле        | обяз. | значение |
|-------------|:----:|----------|
| `key`       | да   | имя поля в строке, откуда берётся значение |
| `label`     | да   | заголовок колонки (и подпись в мобильной карточке) |
| `type`      | нет  | как рисовать ячейку (см. ниже). Без `type` — обычный текст |
| `sort`      | нет  | сделать колонку сортируемой: `"text"` \| `"number"` \| `"date"` |
| `align`     | нет  | `"num"` (число, вправо) \| `"center"` \| без — влево |
| `filterKey` | нет  | связь с фильтром из шапки сайта (см. «Фильтры») |
| `hrefKey`   | для `type:"link"` | имя поля строки, где лежит ссылка |

### Типы ячеек (`type`)

| `type`   | что рисует | что нужно в строке |
|----------|------------|--------------------|
| — (нет)  | обычный текст | `key` → значение |
| `"link"` | ссылка | `key` → текст ссылки, `hrefKey` → адрес |
| `"cat"`  | бейдж категории | `key` → `"a"`/`"b"`/`"c"`/`"d"` (регистр любой; буква покажется заглавной, цвет подставится сам) |

### Строка (`rows[]`)

Ассоциативный массив, ключи = `column.key`. Для `type:"link"` добавьте поле со ссылкой
(имя из `hrefKey`). Пример одной строки:

```json
{
  "employee": "Иванов М.С.", "employeeHref": "/employee?id=42",
  "category": "b",
  "index": "1,31", "checks": 134, "avg": 134,
  "region": "Минск", "brand": "ADEL",
  "pharmacy": "ADEL №14", "pharmacyHref": "/pharmacy?id=14",
  "months": 1
}
```

Числа можно отдавать числом или строкой — сортировка `"number"` понимает и `1234`,
и `"1 234"`, и `"1,31"`.

## Фильтры из шапки сайта

Фильтр в шапке фильтрует таблицу по совпадению **ключей**:
`column.filterKey` в JSON  ↔  ключ поля фильтра в шапке.

Например, поле фильтра «Регион» с ключом `region` отфильтрует колонку, у которой
`"filterKey": "region"`. Поэтому достаточно проставить одинаковые ключи — связывание
произойдёт само, никакого доп. кода не нужно. (Настройка самих полей фильтра — в
[`README.md`](README.md), раздел про шапку.)

Поиск (текстовое поле) ищет по всем колонкам строки.

## Локальная проверка без бэка

Чтобы фронтендер увидел рабочую таблицу без сервера — положите статический JSON в
`public/` и укажите его в `data-table-src`. Готовый пример лежит здесь:

```
public/data/employees.example.json   →  доступен как  /data/employees.example.json
```

```html
<section class="data-panel employees-table" data-data-table
         data-module="DataTable" data-path="components"
         data-table-src="/data/employees.example.json"></section>
```

Формат файла идентичен тому, что отдаёт PHP. На проде просто меняете `data-table-src`
на адрес PHP.

---

## Чек-лист подключения

- [ ] секция с `data-data-table data-module="DataTable" data-path="components"` и `data-table-src="…"`;
- [ ] PHP по этому URL отдаёт `application/json` и **валидный** JSON;
- [ ] на пути к PHP **нет HTTP-авторизации** (иначе браузер не заберёт данные);
- [ ] в `rows` — **все** записи; `columns` описывает каждую колонку;
- [ ] для `type:"link"` у строки есть поле со ссылкой (`hrefKey`);
- [ ] `filterKey` совпадает с ключами полей фильтра в шапке.

> Если таблица когда-нибудь дорастёт до десятков тысяч строк — компонент умеет и
> постраничную подгрузку с сервера (`mode:"server"`, см. `datatable-endpoint.php`).
> Сейчас это **не нужно** — переключение будет точечным, вёрстку менять не придётся.
