# Что обновить на бэке (памятка)

Все правки уже в `main` (`github.com/acrysense/ag`). На стороне бэка нужно одно
основное действие — **пересобрать с правильным `BASE` и перевыложить `dist`**.
Это разом закрывает шрифты, лоадер и фикс фильтров. Ниже детали по каждому пункту.

---

## 0. Главное действие — пересборка и деплой

```bash
nvm use 22
BASE=/bitrix/templates/auth/ npm run build -- --mode cms
```

Выложить **содержимое `dist/`** в корень шаблона `/bitrix/templates/auth/`
(вместе с `manifest.json`, `assets/`, `favicon/`, `fonts/`).

> `BASE` обязателен. Без него `npm run build` берёт дефолт `/`, и все пути
> (шрифты, ленивые чанки) уходят от корня → 404.

`assets.php` должен быть актуальной версии (вход ищется по `name === 'app'`,
`$dir = SITE_TEMPLATE_PATH`) — см. [`assets.php`](assets.php).

---

## 1. Шрифты (была проблема: грузились от корня)

**Причина:** выложенный `app-<hash>.css` был собран без `BASE`, поэтому пути
`@font-face` указывали в корень `/fonts/...`, а файлы лежат в
`/bitrix/templates/auth/fonts/...`.

**Решение:** пересобрать с `BASE=/bitrix/templates/auth/` (см. п.0) и перевыложить
`assets/css/app-<hash>.css` + `manifest.json` (имя файла поменяется — assets.php
подхватит новое по манифесту).

**Как проверить собранное:**
```bash
grep -o '/[^"]*fonts/Inter/Inter-Var\.woff2' dist/assets/css/app-*.css
```
- Правильно: `/bitrix/templates/auth/fonts/Inter/Inter-Var.woff2`
- Если видно `/fonts/Inter/...` — собрано без `BASE`, пересобрать.

То же касается **ленивых чанков** (таблицы, графики, модалки) — без `BASE` они
тоже грузятся от корня и ловят 404.

## 2. Лоадер таблицы (новое, действий не требует)

Пока таблица тянет данные из `data-table-src` (JSON), теперь показывается
спиннер «Загрузка данных…», и панель не схлопывается. На ошибке загрузки —
сообщение «Не удалось загрузить данные».

**От бэка:** ничего, кроме пересборки/деплоя (п.0). Появится автоматически.

---

## 3. Фикс фильтров — дубль кнопок «Применить/Закрыть» (CSS)

На десктопе в выпадашке поля фильтра показывались лишние кнопки
«Применить/Закрыть» (мобильная обвязка) и ломали вёрстку. Поправлено в CSS.

**От бэка:** ничего, кроме пересборки/деплоя (п.0).

---

## 4. Shareable-ссылки для таблиц (фильтры/поиск/сортировка/страница)

Состояние таблицы теперь отражается в URL и восстанавливается при открытии
ссылки, например:
```
?q=ковш&region=Минск&sort=employee&dir=asc&page=2&size=50
```
Можно копировать и слать прямую ссылку на отфильтрованный вид.

**От бэка:** ничего. Фича целиком на фронте, работает в клиентском режиме
(вариант 2 — таблица из JSON). Включается только на страницах с одной таблицей.
На эндпоинты и `data-table-src` не влияет.

---

## Чек-лист после деплоя

- [ ] собрано с `BASE=/bitrix/templates/auth/` (проверка grep по шрифтам из п.1);
- [ ] в `/bitrix/templates/auth/` лежат `manifest.json`, `assets/`, `favicon/`, `fonts/`;
- [ ] `assets.php` актуальной версии (`name === 'app'`, `$dir = SITE_TEMPLATE_PATH`);
- [ ] на странице со списком: грузится лоадер → строится таблица; шрифты на месте;
- [ ] фильтр в шапке: в выпадашке поля нет лишних кнопок (десктоп);
- [ ] фильтры/сортировка/страница отражаются в URL и открываются по ссылке.
