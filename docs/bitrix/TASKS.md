# Подключение задач из JSON — пошагово

Как и таблицы: **бэк отдаёт один JSON со списком задач, фронт строит список сам.**
Клиентский режим (весь список сразу). Задач в панели немного, рендер мгновенный.

---

## Шаг 1. Включить JSON-режим у панели задач

Панель задач (`.tasks-panel`) собирает список из JSON, если на секции задан
`data-tasks-src` — ссылка на PHP, который отдаёт задачи:

```html
<section class="data-panel tasks-panel"
         data-module="TasksPanel" data-path="components"
         data-tasks-src="/local/ajax/tasks.php">
    <!-- шапка, кнопка «Создать», форма, кнопка «Показать завершенные», модалка
         удаления — остаются как в обычной разметке панели;
         сам <ul class="tasks-list"> писать НЕ нужно, фронт построит его -->
</section>
```

Всё остальное (создание, инлайн-редактирование, отметка выполнения, меню «…»)
работает поверх построенного списка автоматически.

> На наших demo-страницах то же включается через партиал:
> `{{> components/TasksPanel/TasksPanel tasksSrc="/data/tasks.example.json" }}`

---

## Шаг 2. PHP отдаёт JSON

Готовый пример: **[`tasks-config.php`](tasks-config.php)**. Должен:
- отдавать `Content-Type: application/json; charset=utf-8`;
- быть доступен по URL из `data-tasks-src` (200, **без HTTP-авторизации**);
- вернуть **все задачи** панели.

---

## Формат JSON (контракт)

```jsonc
{ "tasks": [ /* массив задач */ ] }
```
(можно отдавать и голым массивом `[ ... ]`).

### Задача (`tasks[]`)

| поле             | обяз.        | значение |
|------------------|:-----------:|----------|
| `title`          | да           | текст задачи |
| `assignee`       | да           | ответственный (ФИО) |
| `due`            | да           | срок, `дд.мм.гггг` |
| `hidden`         | нет          | `true` → задача скрыта от сотрудника (значок «глаз») |
| `desc`           | нет          | описание (серый текст под заголовком) |
| `done`           | нет          | `true` → задача выполнена |
| `awaitingConfirm`| нет          | `true` → выполнена, но ждёт подтверждения (кнопка «Подтвердить выполнение», остаётся цветной) |
| `completedBy`    | для `done`   | кто выполнил |
| `completedDate`  | для `done`   | когда выполнил, `дд.мм.гггг` |
| `verifiedBy`     | нет          | кто проверил (строка «Проверена») |
| `verifiedDate`   | нет          | когда проверил |

**Состояния (как рисуется):**
- активная — `done` не задан;
- выполнена, ждёт подтверждения — `done:true` + `awaitingConfirm:true` (+ `completedBy/Date`);
- выполнена и подтверждена — `done:true` (+ `completedBy/Date`) — уходит в «завершённые», приглушается;
- проверена — то же + `verifiedBy/Date`.

Завершённые фронт сам группирует внизу и сворачивает под «Показать завершенные».

### Пример одной задачи

```json
{
  "title": "Выполнить инвентаризацию",
  "assignee": "Иванов М.С.", "due": "05.05.2026",
  "done": true, "awaitingConfirm": true,
  "completedBy": "Петров А.И.", "completedDate": "05.05.2026"
}
```

Полный пример набора — [`public/data/tasks.example.json`](../../public/data/tasks.example.json)
(доступен локально как `/data/tasks.example.json` — можно подставить в `data-tasks-src`
для проверки без бэка).

---

## Пустое состояние

Встроено — если вернуть `{ "tasks": [] }`, панель покажет аккуратную заглушку
**«Нет доступных задач»** (по центру, без строк). Отдельно ничего делать не нужно.

---

## Чек-лист подключения

- [ ] на секции `.tasks-panel` есть `data-tasks-src="…"`, статический `<ul class="tasks-list">` убран;
- [ ] PHP отдаёт `application/json` и валидный JSON `{ "tasks": [...] }`;
- [ ] на пути к PHP нет HTTP-авторизации;
- [ ] у каждой задачи есть `title`, `assignee`, `due`; у выполненных — `completedBy/Date`.

> Создание / редактирование / удаление сейчас demo (без бэка), как и было.
> Когда понадобится их сохранять — это отдельный шаг (POST на бэк из формы).
