# Подключение раздела «Визиты» из JSON — пошагово

Раздел состоит из трёх частей:
1. **План визитов** — календарь (месяц / неделя / день) → JSON визитов (ниже).
2. **План-факт** — таблица → как обычная таблица (`data-table-src`, см. [TABLES.md](TABLES.md)).
3. **История визитов** — таблица → тоже [TABLES.md](TABLES.md).

Таблицы (п.2–3) — обычный `DataTable`, ничего нового: добавляете `data-table-src`
на секцию и отдаёте JSON колонок+строк. Ниже — только про **календарь**.

---

## Календарь: шаг 1 — включить JSON-режим

На элементе календаря задать `data-visits-src` — ссылку на PHP со списком визитов:

```html
<div class="vcal" data-module="VisitsCalendar" data-path="components"
     data-anchor="2026-05-01"
     data-visits-src="/local/ajax/visits.php"></div>
```
- `data-anchor="YYYY-MM-DD"` — какой месяц открыть по умолчанию (любая дата этого месяца).
- Без `data-visits-src` календарь показывает демо-данные (как сейчас).

## Календарь: шаг 2 — PHP отдаёт JSON

Готовый пример: **[`visits-config.php`](visits-config.php)**. Отдаёт
`Content-Type: application/json; charset=utf-8`, **без HTTP-авторизации**, формат:

```jsonc
{ "visits": [ /* массив визитов */ ] }
```
(можно и голым массивом `[ ... ]`).

Отдавайте визиты **минимум за видимый месяц** (лучше с запасом на соседние —
переключение недель/дней внутри месяца работает по этим же данным).

### Визит (`visits[]`)

| поле            | обяз. | значение |
|-----------------|:----:|----------|
| `date`          | да   | дата визита, `дд.мм.гггг` (по ней раскладываются события) |
| `time`          | да   | время, `чч:мм` (для недельной/дневной сетки и сортировки) |
| `name`          | да   | сотрудник (ФИО) |
| `cat`           | да   | категория `a`/`b`/`c`/`d` — цвет метки |
| `pharmacy`      | да   | аптека (адрес/название одной строкой) |
| `phone`         | нет  | телефон сотрудника (в попапе) |
| `manager`       | нет  | менеджер (в попапе) |
| `managerPhone`  | нет  | телефон менеджера |
| `managerEmail`  | нет  | email менеджера |
| `type`          | нет  | тип визита (напр. «Коучинг») |
| `comment`       | нет  | комментарий |
| `status`        | нет  | `"planned"` (по умолчанию) или `"confirmed"` — определяет вид попапа |
| `confirmedDate` | для `confirmed` | когда подтверждён, напр. `09.03.2026, 12:24` |
| `coords`        | нет  | координаты (в попапе подтверждённого) |
| `checklist`     | нет  | результат чек-листа, напр. `78 / 100` |
| `cancelled`     | нет  | `true` → визит отменён (метка зачёркнута) |

### Пример одного визита

```json
{
  "date": "05.05.2026", "time": "09:00",
  "name": "Медицинская Т.В.", "cat": "b",
  "pharmacy": "Могилев, ADEL, Аптека №12",
  "manager": "Петров А.И.", "managerPhone": "+375 29 123-10-10",
  "type": "Коучинг", "comment": "Обсудить показатели продаж СТМ.",
  "status": "confirmed", "confirmedDate": "09.03.2026, 12:24",
  "coords": "5534444.44 , 455543.22", "checklist": "78 / 100"
}
```

Локальный пример набора — [`public/data/visits.example.json`](../../public/data/visits.example.json)
(доступен как `/data/visits.example.json` — можно подставить в `data-visits-src`).

## Пустое / загрузка
- Пока JSON грузится — в календаре спиннер «Загрузка визитов…».
- Нет визитов в месяце — календарь просто пустой (ячейки без событий).
- Кнопка «Создать визит» и попапы событий работают поверх этих данных.

> Создание/изменение визита сейчас demo (модалка без сохранения на бэк) — POST это
> отдельный шаг.

## Чек-лист
- [ ] у `.vcal` есть `data-visits-src` (и `data-anchor` на нужный месяц);
- [ ] таблицы «План-факт» / «История» подключены через `data-table-src` (см. TABLES.md);
- [ ] PHP отдаёт `application/json` без авторизации;
- [ ] у визита есть `date`, `time`, `name`, `cat`, `pharmacy`.
