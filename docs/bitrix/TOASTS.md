# Тосты (всплывающие уведомления) — как вызывать

Контейнер тостов уже на **каждой** странице (в футере), отдельно подключать не нужно.
Есть три способа показать тост — для Bitrix удобнее всего **способ 1**.

Тип: `success` (зелёная галка, по умолчанию) или `error` (красный треугольник).

---

## Способ 1 — серверная разметка (рекомендуется для Bitrix)

Просто выведите **где угодно на странице** скрытый `<div data-toast …>` — фронт покажет
тост при загрузке и сам удалит этот элемент. Идеально после действия (сохранил → отдал
страницу с этим блоком).

```html
<div data-toast
     data-type="error"
     data-title="Ошибка"
     data-text="Не удалось сохранить задачу"
     data-timeout="5000"></div>
```

| атрибут | значение |
|---|---|
| `data-type` | `success` \| `error` (по умолчанию `success`) |
| `data-title` | заголовок (можно опустить) |
| `data-text` | текст (можно опустить) |
| `data-timeout` | мс до автозакрытия, по умолчанию `5000`; `0` → не закрывать автоматически |

Можно вывести **несколько** блоков `data-toast` — покажутся стопкой (максимум 5 на экране).

---

## Способ 2 — из JavaScript (глобальная функция `window.toast`)

```js
window.toast.success('Задача сохранена')
window.toast.error('Ошибка при сохранении')

// с заголовком / настройками:
window.toast.show({ type: 'error', title: 'Ошибка', text: 'Проверьте поля', timeout: 8000 })
```
- `success(text, opts?)` / `error(text, opts?)` / `show({ type, title, text, timeout })`.

---

## Способ 3 — событие (если `window.toast` ещё не готов)

```js
document.dispatchEvent(new CustomEvent('toast:show', {
  detail: { type: 'error', title: 'Ошибка', text: 'Сессия истекла' }
}))
```

---

## Заметки
- Текст и заголовок экранируются автоматически — HTML внутри не вставится.
- Тост закрывается по крестику, по таймауту, наведение мышью — пауза таймера.
- Контейнер живёт в футере (`{{> components/Toasts/Toasts}}`) — есть на всех страницах.
